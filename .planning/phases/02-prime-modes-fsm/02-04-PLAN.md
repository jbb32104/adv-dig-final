Create `tb/mode_fsm_tb.v` with these exact specifications:

**Header and timescale:**
```verilog
`timescale 1ns / 1ps
// Self-checking integration testbench for mode_fsm.v + prime_engine.v + elapsed_timer.v + prime_accumulator.v
// Tests: Mode 1 (N=100, expect 25 primes), Mode 2 (T=3 sim-seconds), Mode 3 (prime + composite)
```

**Clock generation:** 10ns period (100 MHz), `always #5 clk = ~clk;`

**Module instantiation architecture:**
The testbench instantiates ALL modules and wires them together, mirroring the future top-level:

1. `mode_fsm #(.WIDTH(27)) u_fsm (...)` — connect all ports
2. `prime_engine #(.WIDTH(27)) u_eng (.clk, .rst, .start(eng_start_w), .candidate(eng_candidate_w), .done_ff(eng_done_w), .is_prime_ff(eng_is_prime_w), .busy_ff(eng_busy_w))`
3. `elapsed_timer #(.TICK_PERIOD(100)) u_timer (.clk, .rst, .freeze(timer_freeze_w), .cycle_count_ff(cycle_count_w), .seconds_ff(seconds_w), .second_tick_ff())`
4. `prime_accumulator #(.WIDTH(27), .FIFO_DEPTH(32)) u_acc (.clk, .rst, .prime_valid(prime_valid_w), .prime_data(prime_data_w), .prime_fifo_rd_en(1'b0), ...)`

**Wire names for inter-module connections:**
- `wire eng_start_w, eng_done_w, eng_is_prime_w, eng_busy_w;`
- `wire [26:0] eng_candidate_w;`
- `wire prime_valid_w; wire [26:0] prime_data_w; wire prime_fifo_full_w;`
- `wire timer_freeze_w; wire [31:0] seconds_w, cycle_count_w;`
- `wire done_w; wire is_prime_result_w; wire [3:0] state_out_w;`
- All 20 `last20_*_w` wires

**Connect mode_fsm outputs to sub-module inputs:**
- `u_fsm.eng_start_ff` → `u_eng.start`
- `u_fsm.eng_candidate_ff` → `u_eng.candidate`
- `u_eng.done_ff` → `u_fsm.eng_done_ff`
- `u_eng.is_prime_ff` → `u_fsm.eng_is_prime_ff`
- `u_eng.busy_ff` → `u_fsm.eng_busy_ff`
- `u_fsm.prime_valid_ff` → `u_acc.prime_valid`
- `u_fsm.prime_data_ff` → `u_acc.prime_data`
- `u_acc.prime_fifo_full_ff` → `u_fsm.prime_fifo_full_ff`
- `u_fsm.timer_freeze_ff` → `u_timer.freeze`
- `u_timer.seconds_ff` → `u_fsm.seconds_ff`
- `u_timer.cycle_count_ff` → `u_fsm.cycle_count_ff`

**Tie off unused ports:**
- `u_acc.prime_fifo_rd_en = 1'b0` (no DDR2 reader in Phase 2, per D-05)

**Testbench input registers:**
- `reg [1:0] mode_sel; reg [26:0] n_limit; reg [31:0] t_limit; reg [26:0] check_candidate; reg go;`

**Error counting:**
```verilog
integer error_count;
initial error_count = 0;
```

**Test sequence:**

**Test 1: Mode 1, N=100 — find all primes <= 100 (PRIME-02, PRIME-05)**
1. Reset (rst=1 for 20ns, then rst=0, wait 2 cycles)
2. Set `mode_sel = 2'd1; n_limit = 27'd100; go = 1;` wait one cycle, `go = 0;`
3. Wait for `done_w == 1` with a timeout guard: `repeat (500000) @(posedge clk); if (!done_w) $fatal;`
   (N=100 should complete in ~50,000 cycles; 500,000 is generous timeout)
4. Check `u_acc.prime_count_ff == 25` (25 primes <= 100)
5. Check `timer_freeze_w == 1` (timer frozen)
6. Check `done_w == 1`
7. Verify last-20 ring buffer: the last 5 of the 25 primes (71, 73, 79, 83, 89, 97) should appear in the last20 outputs. Check at least last20 contains 97 (the largest prime <= 100).
8. Record `cycle_count_w` value; wait 10 more cycles; verify `cycle_count_w` has NOT changed (frozen).

**Test 2: Mode 2, T=3 sim-seconds (PRIME-03, PRIME-06)**
1. Pulse reset again to clear state
2. Set `mode_sel = 2'd2; t_limit = 32'd3; go = 1;` wait one cycle, `go = 0;`
3. Wait for `done_w == 1` with timeout: `repeat (500) @(posedge clk);`
   (With TICK_PERIOD=100, 3 seconds = 300 cycles; timeout at 500 is generous)
4. Check `seconds_w >= 3` (timer reached at least T)
5. Check `u_acc.prime_count_ff > 0` (at least some primes found)
6. Check `timer_freeze_w == 1`
7. Record `seconds_w`; wait 10 cycles; verify `seconds_w` unchanged (frozen).
8. $display the prime count found for informational purposes.

**Test 3: Mode 3, candidate=97 (prime) (PRIME-04)**
1. Pulse reset
2. Set `mode_sel = 2'd3; check_candidate = 27'd97; go = 1;` wait one cycle, `go = 0;`
3. Wait for `done_w == 1` with timeout (5000 cycles)
4. Check `is_prime_result_w == 1` (97 is prime)
5. Check `timer_freeze_w == 1`

**Test 4: Mode 3, candidate=99 (composite) (PRIME-04)**
1. Pulse reset
2. Set `mode_sel = 2'd3; check_candidate = 27'd99; go = 1;` wait one cycle, `go = 0;`
3. Wait for `done_w == 1` with timeout (5000 cycles)
4. Check `is_prime_result_w == 0` (99 = 9 * 11, composite)
5. Check `timer_freeze_w == 1`

**Test 5: Mode 3, candidate=2 (edge case — smallest prime)**
1. Pulse reset
2. Set `mode_sel = 2'd3; check_candidate = 27'd2; go = 1;` wait one cycle, `go = 0;`
3. Wait for done, check `is_prime_result_w == 1`

**Final verdict:**

**Important implementation notes:**
- All inter-module signals use `wire` declarations, not `reg`. The mode_fsm output ports (which are `reg` inside mode_fsm) drive wires in the testbench that connect to sub-module inputs.

- Use `@(posedge clk)` for all timing; no `#` delays except in clock generation.
- Timeout guards prevent infinite loops if the FSM hangs.
