// Self-checking unit testbench for prime_accumulator.v and elapsed_timer.v

- Instance name: `u_acc`

**Helper task for errors:**
```verilog
integer error_count;
initial error_count = 0;

task check;
    input [255:0] test_name;
    input [31:0]  actual;
    input [31:0]  expected;
    begin
        if (actual !== expected) begin
            $display("FAIL: %0s — got %0d, expected %0d at time %0t", test_name, actual, expected, $time);

        end
    end

```

**Test sequence (in `initial` block):**

1. **Reset:** Assert rst=1 for 20ns, then rst=0. Wait 2 cycles.

2. **Test A: elapsed_timer basic counting**
   - Let timer run for 50 cycles (no freeze)
   - Check `cycle_count_ff == 50` (approximately, accounting for reset cycles)
   - Let timer run to 100+ cycles total to trigger one second_tick
   - Check `seconds_ff == 1`

3. **Test B: elapsed_timer freeze**
   - Record current `cycle_count_ff` and `seconds_ff` values
   - Assert `freeze = 1`
   - Wait 20 cycles
   - Check `cycle_count_ff` has not changed from recorded value
   - Check `seconds_ff` has not changed from recorded value
   - Deassert freeze, verify counting resumes

4. **Test C: prime_accumulator — write 5 primes, check count and FIFO**
   - Feed 5 primes (2, 3, 5, 7, 11) via prime_valid/prime_data pulses (one per clock cycle, with prime_valid high for exactly one cycle each)
   - Check `prime_count_ff == 5`
   - Check `prime_fifo_empty_ff == 0`
   - Check `prime_fifo_full_ff == 0`

5. **Test D: prime_accumulator — read back from FIFO**
   - Assert `prime_fifo_rd_en = 1` for 5 cycles
   - After one-cycle read latency, check that data comes back in order: 2, 3, 5, 7, 11
   - After 5 reads, check `prime_fifo_empty_ff == 1`
   - Deassert rd_en

6. **Test E: prime_accumulator — fill FIFO to full (32 entries)**
   - Write 32 primes (use values 100, 101, 102, ..., 131)
   - After 32 writes, check `prime_fifo_full_ff == 1`
   - Check `prime_count_ff == 37` (5 from test C + 32 from test E)
   - Attempt one more write — prime_count should NOT increment (FIFO full blocks write)

7. **Test F: last-20 ring buffer wrap**
   - After the 37 total primes written (5 + 32), the last 20 should be values 117..131 and the 5 primes that wrapped. Actually, since test E wrote 32 with FIFO full blocking on the 33rd, total successful writes = 5 + 32 = 37.
   - The ring buffer's last 20 entries should be the 20 most recently written primes.
   - Read last20_0_ff through last20_19_ff and verify they contain the correct 20 values (the most recent 20 of the 37 written primes).
   - Check the ring buffer wrap: with 37 writes, wr_ptr wraps through 20 almost twice. The last 20 written values are primes #18 through #37 (0-indexed: 17 through 36).

8. **Test G: FIFO drain and re-fill**
   - Read all 32 entries from FIFO (assert rd_en for 32 cycles)
   - Verify `prime_fifo_empty_ff == 1` after drain
   - Write 3 more primes, verify `prime_fifo_empty_ff == 0`, `prime_count_ff == 40`

9. **Final verdict:**

if (error_count == 0) begin

end else begin

    $fatal;
end
$finish;
