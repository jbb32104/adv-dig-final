Create `rtl/elapsed_timer.v` with these exact specifications:

**Module declaration (ANSI ports):**
```verilog
module elapsed_timer #(
    parameter TICK_PERIOD = 100_000_000
) (
    input  wire        clk,
    input  wire        rst,
    input  wire        freeze,
    output reg  [31:0] cycle_count_ff,
    output reg  [31:0] seconds_ff,
    output reg         second_tick_ff
);
```

**Internal registers:**
- `localparam TICK_BITS = 27;` (2^27 = 134,217,728 > 100,000,000)
- `reg [TICK_BITS-1:0] tick_cnt_ff;` — counts 0 to TICK_PERIOD-1

**Combinational next-state signals (all declared as `reg`, used with `=` only):**
- `next_tick_cnt`, `next_cycle_count`, `next_seconds`, `next_second_tick`

**Combinational block (`always @(*)`):**
1. Defaults: hold all `next_` = current `_ff` values; `next_second_tick = 1'b0` (pulse default off)
2. `if (rst)` — clear all counters to zero (per D-08)
3. `else if (freeze)` — hold everything, highest priority after reset (per D-11, Pitfall 6 in RESEARCH.md)
4. `else` — increment `next_cycle_count = cycle_count_ff + 32'd1`; check `tick_cnt_ff == TICK_PERIOD - 1` (use Verilog integer literal comparison, not sized cast — iverilog does not support `TICK_BITS'(...)` syntax); if match: reset tick_cnt to 0, increment seconds, pulse second_tick; else: increment tick_cnt
5. No `case` statement needed — pure if-else chain with final `else`

**Flop block (`always @(posedge clk)`):**
- `tick_cnt_ff <= next_tick_cnt;`
- `cycle_count_ff <= next_cycle_count;`
- `seconds_ff <= next_seconds;`
- `second_tick_ff <= next_second_tick;`

**Important iverilog compatibility note:** Do NOT use the `TICK_BITS'(TICK_PERIOD - 1)` sized literal syntax from the RESEARCH.md skeleton. iverilog does not support parameterized bit-width casts. Instead use plain integer comparison: `if (tick_cnt_ff == TICK_PERIOD - 1)`.

Per D-08: runs on 100 MHz system clock only — no PLL or separate clock domain.
Per D-09: 27-bit cycle counter resets at TICK_PERIOD, asserting one-cycle second_tick pulse.
Per D-10: cycle_count_ff (32-bit, every clock), seconds_ff (32-bit, every second_tick).
Per D-11: freeze halts both counters immediately.

Create `rtl/prime_accumulator.v` with these exact specifications:

**Module declaration (ANSI ports):**
```verilog
module prime_accumulator #(
    parameter WIDTH      = 27,
    parameter FIFO_DEPTH = 32
) (
    input  wire             clk,
    input  wire             rst,
    // Write interface (from mode_fsm)
    input  wire             prime_valid,
    input  wire [WIDTH-1:0] prime_data,
    // FIFO read interface (for Phase 3 DDR2 writer)
    input  wire             prime_fifo_rd_en,
    output reg  [WIDTH-1:0] prime_fifo_rd_data_ff,
    output reg              prime_fifo_empty_ff,
    output reg              prime_fifo_full_ff,
    // Live status outputs
    output reg  [31:0]      prime_count_ff,
    // Last-20 ring buffer read interface
    output reg  [WIDTH-1:0] last20_0_ff,  last20_1_ff,  last20_2_ff,  last20_3_ff,
    output reg  [WIDTH-1:0] last20_4_ff,  last20_5_ff,  last20_6_ff,  last20_7_ff,
    output reg  [WIDTH-1:0] last20_8_ff,  last20_9_ff,  last20_10_ff, last20_11_ff,
    output reg  [WIDTH-1:0] last20_12_ff, last20_13_ff, last20_14_ff, last20_15_ff,
    output reg  [WIDTH-1:0] last20_16_ff, last20_17_ff, last20_18_ff, last20_19_ff
);
```

**IMPORTANT on last20 ports:** Use 20 individual `output reg` ports (last20_0_ff through last20_19_ff) rather than a reg array output, because Verilog-2001 does not allow arrays in port declarations. Internally, use a reg array `last20_ff[0:19]` and copy to the output ports in the flop block.

**Internal constants:**
- `localparam PTR_W = 5;` (log2(32) = 5)
- `localparam RING_SIZE = 20;`

**Internal registers (_ff suffix):**
- `reg [WIDTH-1:0] fifo_mem [0:FIFO_DEPTH-1];` — FIFO storage (BRAM inference)
- `reg [PTR_W-1:0] fifo_wr_ptr_ff;` — FIFO write pointer
- `reg [PTR_W-1:0] fifo_rd_ptr_ff;` — FIFO read pointer
- `reg [PTR_W:0]   fifo_count_ff;` — one extra bit to distinguish full/empty (6 bits for depth 32)
- `reg [WIDTH-1:0] last20_ff [0:19];` — ring buffer storage
- `reg [4:0]       ring_wr_ptr_ff;` — ring buffer write pointer (0..19)

**Combinational signals (next_ prefix, blocking `=`):**
- next_fifo_wr_ptr, next_fifo_rd_ptr, next_fifo_count, next_prime_count, next_ring_wr_ptr
- next_fifo_empty, next_fifo_full

**Combinational block (`always @(*)`):**
1. Defaults: hold all next_ = current _ff values
2. `if (rst)` — clear all pointers, count, empty=1, full=0, prime_count=0
3. `else begin` — main logic:

   **FIFO full/empty flags (per D-06, Pitfall 4):**
   - `next_fifo_empty = (next_fifo_count == 0);` (update based on computed next_fifo_count below)
   - `next_fifo_full  = (next_fifo_count == FIFO_DEPTH);`

   **FIFO write (when prime_valid && !fifo_full_ff):**
   - Increment fifo_wr_ptr (wraps naturally at PTR_W bits since FIFO_DEPTH is power-of-2)
   - Increment fifo_count

   **FIFO read (when prime_fifo_rd_en && !fifo_empty_ff):**
   - Increment fifo_rd_ptr (wraps naturally)
   - Decrement fifo_count

   **Simultaneous read+write:** count stays same (increment and decrement cancel)

   **Prime count (per D-07):**
   - `if (prime_valid && !fifo_full_ff)` → `next_prime_count = prime_count_ff + 32'd1`

   **Ring buffer write pointer (per D-07, Pattern 4):**
   - `if (prime_valid && !fifo_full_ff)` → if `ring_wr_ptr_ff == 5'd19` then `next_ring_wr_ptr = 5'd0` else `next_ring_wr_ptr = ring_wr_ptr_ff + 5'd1`

   Compute next_fifo_empty and next_fifo_full AFTER updating next_fifo_count.

**Flop block (`always @(posedge clk)`):**
- All pointer/count _ff <= next_ assignments
- `prime_count_ff <= next_prime_count;`
- `ring_wr_ptr_ff <= next_ring_wr_ptr;`
- `prime_fifo_empty_ff <= next_fifo_empty;`
- `prime_fifo_full_ff <= next_fifo_full;`

**Separate FIFO write flop block:**
```verilog
always @(posedge clk) begin
    if (prime_valid && !fifo_full_ff) begin
        fifo_mem[fifo_wr_ptr_ff] <= prime_data;
    end
end
```

**Separate FIFO read flop block (registered read for BRAM inference):**
```verilog
always @(posedge clk) begin
    if (prime_fifo_rd_en && !prime_fifo_empty_ff) begin
        prime_fifo_rd_data_ff <= fifo_mem[fifo_rd_ptr_ff];
    end
end
```

**Ring buffer write flop block:**
```verilog
always @(posedge clk) begin
    if (rst) begin
        // Reset all 20 entries explicitly (no for loop!)
        last20_ff[0]  <= {WIDTH{1'b0}};
        last20_ff[1]  <= {WIDTH{1'b0}};
        // ... through last20_ff[19]
    end else if (prime_valid && !fifo_full_ff) begin
        last20_ff[ring_wr_ptr_ff] <= prime_data;
    end
end
```

**Last20 output copy flop block:**
```verilog
always @(posedge clk) begin
    last20_0_ff  <= last20_ff[0];
    last20_1_ff  <= last20_ff[1];
    // ... through last20_19_ff <= last20_ff[19];
end
```

Note: Multiple `always @(posedge clk)` blocks are acceptable per class rules as long as each register is written in exactly one block. The main comb block has one `always @(*)`. The flop blocks are split by function for clarity (FIFO write, FIFO read, ring buffer, main pointers). This matches the divider.v pattern where the main logic is in one comb block.

Per D-04: BRAM-based FIFO buffering found primes.
Per D-05: FIFO read-side ports: prime_fifo_rd_en, prime_fifo_rd_data_ff, prime_fifo_empty_ff.
Per D-06: FIFO full stalls mode_fsm (via prime_fifo_full_ff output).
Per D-07: prime_count_ff (32-bit running count), last20 ring buffer (20 entries).
