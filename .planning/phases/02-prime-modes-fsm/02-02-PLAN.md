Create `rtl/mode_fsm.v` with these exact specifications:

**Module declaration (ANSI ports):**
```verilog
module mode_fsm #(
    parameter WIDTH = 27
) (
    input  wire             clk,
    input  wire             rst,
    // User interface (directly driven by testbench in Phase 2; joystick/7SD in Phase 4)
    input  wire [1:0]       mode_sel,       // 2'd1=Mode1, 2'd2=Mode2, 2'd3=Mode3
    input  wire [WIDTH-1:0] n_limit,        // Mode 1: find primes <= N
    input  wire [31:0]      t_limit,        // Mode 2: run for T seconds
    input  wire [WIDTH-1:0] check_candidate,// Mode 3: single number to test
    input  wire             go,             // pulse to start selected mode
    // prime_engine interface
    output reg              eng_start_ff,
    output reg  [WIDTH-1:0] eng_candidate_ff,
    input  wire             eng_done_ff,
    input  wire             eng_is_prime_ff,
    input  wire             eng_busy_ff,
    // prime_accumulator interface
    output reg              prime_valid_ff,
    output reg  [WIDTH-1:0] prime_data_ff,
    input  wire             prime_fifo_full_ff,
    // elapsed_timer interface
    output reg              timer_freeze_ff,
    input  wire [31:0]      seconds_ff,
    input  wire [31:0]      cycle_count_ff,
    // Status outputs
    output reg              done_ff,
    output reg              is_prime_result_ff,
    output reg  [3:0]       state_out_ff     // expose FSM state for debug/testbench
);
```

**State encoding (4-bit binary, per discretion — binary preferred):**
```verilog
localparam [3:0]
    IDLE          = 4'd0,
    MODE_SELECT   = 4'd1,
    NUMBER_ENTRY  = 4'd2,
    TIME_ENTRY    = 4'd3,
    PRIME_RUN     = 4'd4,
    PRIME_DONE    = 4'd5,
    ISPRIME_ENTRY = 4'd6,
    ISPRIME_RUN   = 4'd7,
    ISPRIME_DONE  = 4'd8;
```

**Internal registers (_ff suffix):**
- `reg [3:0] state_ff;`
- `reg [1:0] mode_sel_ff;` — latched mode selection
- `reg [WIDTH-1:0] n_limit_ff;` — latched N limit for Mode 1
- `reg [31:0] t_limit_ff;` — latched T limit for Mode 2 (per D-13)
- `reg [WIDTH-1:0] candidate_ff;` — current candidate being fed to prime_engine
- `reg step_toggle_ff;` — 0 = next step +2, 1 = next step +4 (per D-01)
- `reg [1:0] init_phase_ff;` — 2'b10 = feed 2, 2'b01 = feed 3, 2'b00 = main loop (per Pattern 3 in RESEARCH)
- `reg waiting_result_ff;` — 1 when prime_engine is running and we're waiting for done_ff

**Combinational signals (next_ prefix, blocking `=`):**
All corresponding `next_` versions of the above registers, plus `next_eng_start`, `next_prime_valid`, `next_prime_data`, `next_timer_freeze`, `next_done`, `next_is_prime_result`, `next_state_out`.

**Combinational block (`always @(*)`) — the core FSM logic:**

1. **Defaults:** Hold all `next_` = current `_ff`; set pulse signals to 0:
   - `next_eng_start = 1'b0;`
   - `next_prime_valid = 1'b0;`
   - `next_done = done_ff;` (done holds high in DONE states, per RESEARCH Open Question 3)
   - `next_timer_freeze = timer_freeze_ff;`

2. **`if (rst)`** — clear everything: state to IDLE, all registers to 0, done=0, freeze=0.

3. **`else begin case (state_ff)`** — state machine:

   **IDLE:** If `go` is pulsed, latch `mode_sel` into `mode_sel_ff` and transition to MODE_SELECT. Else hold IDLE.

   **MODE_SELECT:** Based on `mode_sel_ff`:
   - `2'd1` (Mode 1): latch `n_limit` into `n_limit_ff`, transition to NUMBER_ENTRY
   - `2'd2` (Mode 2): latch `t_limit` into `t_limit_ff`, transition to TIME_ENTRY
   - `2'd3` (Mode 3): latch `check_candidate` into `candidate_ff`, transition to ISPRIME_ENTRY
   - `default`: return to IDLE

   **NUMBER_ENTRY:** Initialize for Mode 1 candidate enumeration:
   - `next_init_phase = 2'b10;` (start with candidate 2)
   - `next_candidate = {{WIDTH-2{1'b0}}, 2'd2};` (candidate = 2)
   - `next_step_toggle = 1'b0;`
   - `next_waiting_result = 1'b0;`
   - `next_done = 1'b0;`
   - `next_timer_freeze = 1'b0;`
   - Transition to PRIME_RUN

   **TIME_ENTRY:** Initialize for Mode 2 candidate enumeration (same as Mode 1 init):
   - Same init_phase, candidate, step_toggle, waiting_result setup as NUMBER_ENTRY
   - `next_done = 1'b0;`
   - `next_timer_freeze = 1'b0;`
   - Transition to PRIME_RUN

   **PRIME_RUN:** This is the main computation loop for Modes 1 and 2.

   Priority order in if-else chain (per RESEARCH Mode 2 Termination pattern):

   a. **Mode 2 timeout check (HIGHEST PRIORITY per D-12):**
      `if (mode_sel_ff == 2'd2 && seconds_ff >= t_limit_ff)` → transition to PRIME_DONE, assert freeze, assert done.
      This fires even if prime_engine is mid-computation — per D-12, terminate immediately.

   b. **Mode 1 completion check:**
      `else if (mode_sel_ff == 2'd1 && !waiting_result_ff && candidate_ff > n_limit_ff)` → transition to PRIME_DONE, assert freeze, assert done.
      Only check when not waiting for a result (all candidates up to N have been submitted and their results collected).

   c. **Waiting for prime_engine result:**
      `else if (waiting_result_ff)` → check `eng_done_ff`:
      - If `eng_done_ff`:
        - `next_waiting_result = 1'b0;`
        - If `eng_is_prime_ff && !prime_fifo_full_ff`: pulse `next_prime_valid = 1'b1;` with `next_prime_data = candidate_ff;`
        - Advance candidate per D-01/D-02:
          - If `init_phase_ff == 2'b10`: `next_init_phase = 2'b01; next_candidate = {{WIDTH-2{1'b0}}, 2'd3};` (next is 3)
          - Else if `init_phase_ff == 2'b01`: `next_init_phase = 2'b00; next_candidate = {{WIDTH-3{1'b0}}, 3'd5}; next_step_toggle = 1'b0;` (next is 5, toggle=0 means +2 next)
          - Else (main loop): if `step_toggle_ff == 1'b0` then `next_candidate = candidate_ff + {{WIDTH-2{1'b0}}, 2'd2}; next_step_toggle = 1'b1;` else `next_candidate = candidate_ff + {{WIDTH-3{1'b0}}, 3'd4}; next_step_toggle = 1'b0;`
      - If not `eng_done_ff`: hold in PRIME_RUN, stay waiting

   d. **FIFO full stall (per D-06):**
      `else if (prime_fifo_full_ff)` → hold, do not start engine. `next_eng_start = 1'b0;`

   e. **Ready to start next candidate:**
      `else if (!eng_busy_ff)` → `next_eng_start = 1'b1; next_eng_candidate = candidate_ff; next_waiting_result = 1'b1;`
      (per Pitfall 1: gate on ~eng_busy_ff, NOT on eng_done_ff)

   f. **else:** hold in PRIME_RUN (engine busy, not done yet — shouldn't normally reach here)

   **PRIME_DONE:** Hold done_ff=1, timer_freeze=1. If `go` pulsed, transition back to IDLE (allow re-run). Else hold.

   **ISPRIME_ENTRY:** Initialize for Mode 3:
   - candidate_ff already latched in MODE_SELECT
   - `next_waiting_result = 1'b0;`
   - `next_done = 1'b0;`
   - `next_timer_freeze = 1'b0;`
   - Transition to ISPRIME_RUN

   **ISPRIME_RUN:** Single prime_engine invocation:
   - If `waiting_result_ff && eng_done_ff`:
     - `next_is_prime_result = eng_is_prime_ff;`
     - If prime: pulse `next_prime_valid = 1'b1; next_prime_data = candidate_ff;`
     - Transition to ISPRIME_DONE, assert freeze, assert done
   - Else if `waiting_result_ff && !eng_done_ff`: hold (waiting)
   - Else if `!eng_busy_ff`: start engine: `next_eng_start = 1'b1; next_eng_candidate = candidate_ff; next_waiting_result = 1'b1;`
   - Else: hold

   **ISPRIME_DONE:** Hold done_ff=1, timer_freeze=1, is_prime_result_ff holds. If `go` pulsed, transition to IDLE. Else hold.

   **default:** `next_state = IDLE;`

4. **After the case/endcase:** `next_state_out = next_state;` (expose state for debug)

**Flop block (`always @(posedge clk)`):**
All `_ff <= next_` assignments:
- state_ff, mode_sel_ff, n_limit_ff, t_limit_ff, candidate_ff, step_toggle_ff, init_phase_ff, waiting_result_ff
- eng_start_ff, eng_candidate_ff, prime_valid_ff, prime_data_ff, timer_freeze_ff
- done_ff, is_prime_result_ff, state_out_ff

**Critical timing notes (from RESEARCH Pitfalls):**
- Pitfall 1: Gate eng_start on `~eng_busy_ff`, NOT on `eng_done_ff`.
- Pitfall 2: Candidate advance happens when eng_done_ff fires; eng_start fires on the NEXT cycle when eng_busy_ff is low and the new candidate is already in candidate_ff.
- Pitfall 3: Mode 2 timeout transitions immediately to PRIME_DONE even if engine is mid-computation.
