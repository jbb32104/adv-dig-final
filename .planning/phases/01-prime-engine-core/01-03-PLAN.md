Perform a systematic audit of `rtl/divider.v` and `rtl/prime_engine.v` against each INFRA rule. For each rule, run the specified check, identify violations, and fix them in-place.

**INFRA-03: _ff suffix for FFs; _n suffix for active-low signals**

Check: Every `reg` that appears on the left side of a `<=` assignment inside `always @(posedge clk)` MUST have `_ff` suffix. Every signal that is active-low MUST have `_n` suffix.

Audit steps:
1. List all `reg` declarations in each file.
2. For each reg, check if it is driven by `<=` inside `always @(posedge clk)`.
3. If yes, verify the signal name ends with `_ff`.
4. Check for any active-low signals (active when 0) -- these must end with `_n`.
5. Fix: rename any non-compliant signals throughout the file (declaration, all usages, port connections).

**INFRA-04: Blocking/non-blocking separation**

Check: No `always` block contains BOTH `=` and `<=` assignments.

Audit steps:
1. For each `always @(posedge clk)` block: verify ONLY `<=` appears (no `=` assignments to signals; `=` for local variables like loop counters in TB is OK but should not appear in synthesis always blocks).
2. For each `always @(*)` block: verify ONLY `=` appears (no `<=`).
3. Fix: move any misplaced assignments to the correct block.

**INFRA-05: No for loops in synthesis files**

Check: `grep -n "for\s*(" rtl/divider.v rtl/prime_engine.v` must return empty.

Audit steps:
1. Run the grep.
2. If any matches, refactor the for loop into FSM state transitions or unrolled logic.

**INFRA-06: Combinational in always @(*); FFs in always @(posedge clk)**

Check: No combinational logic (next-state computation, output generation) lives in the posedge clk block. No flip-flop updates live in the always @(*) block.

Audit steps:
1. Verify `always @(posedge clk)` blocks contain ONLY `<=` assignments to `_ff` registers (and reset logic).
2. Verify `always @(*)` blocks contain ONLY `=` assignments computing next-state/output values.
3. Verify NO `always @(posedge clk or ...)` patterns exist (async reset forbidden).
4. Fix: restructure any violations.

**INFRA-07: default in all case; final else in all if-else**

Check: Every `case` statement has a `default:` branch. Every `if` that has an `else if` also has a final `else`.

Audit steps:
1. Count `case` statements: `grep -c "case\s*(" rtl/divider.v rtl/prime_engine.v`
2. Count `default:` statements: `grep -c "default:" rtl/divider.v rtl/prime_engine.v`
3. These counts must match (one default per case).
4. For if-else chains: run `grep -n "else if" rtl/divider.v rtl/prime_engine.v` and for EACH match, manually verify there is a trailing `else` (not `else if`) that closes the chain. This requires reading the surrounding code context for each hit, not just counting. A standalone `if`/`else` pair (no `else if`) is fine and does not need additional inspection.
5. Fix: add missing `default:` or `else` branches.

**Additional checks:**
- ANSI module port declarations (not old-style port lists): verify `module name #(...) (input wire ..., output reg ...)` format.
- No `integer` type in synthesis files (use `reg [N:0]` with explicit width).
- No `$display`, `$finish`, or other simulation-only constructs in synthesis files.
- `rst` is synchronous: appears inside `always @(posedge clk)` as `if (rst)`, NOT in sensitivity list.

**After all fixes:** Re-read both files to confirm all changes are consistent (no dangling references to renamed signals).
