  <action>
Create `tb/prime_engine_tb.v` -- a self-checking testbench that sweeps candidates 2 through 10007.

**Module declaration:**
```
`timescale 1ns/1ps
module prime_engine_tb;
```

**Parameters and DUT signals:**
```
parameter WIDTH = 27;
parameter CLK_PERIOD = 10;  // 100 MHz
parameter MAX_CANDIDATE = 10007;
parameter TIMEOUT_CYCLES = 200000;

reg             clk;
reg             rst;
reg             start;
reg [WIDTH-1:0] candidate;
wire            done_ff;
wire            is_prime_ff;
wire            busy_ff;
```

**Golden list memory with load guard:**
```
reg golden [0:MAX_CANDIDATE];
initial begin
    $readmemb("tb/golden_primes.mem", golden);
    // GUARD: iVerilog silently zeroes memory on missing file.
    // golden[2] must be 1 (2 is prime). If it is 0, the file did not load.
    if (golden[2] !== 1'b1) begin
        $display("FATAL: golden_primes.mem not loaded or path wrong");
        $finish;
    end
end
```
IMPORTANT: The `$readmemb` path is relative to the directory where `vvp` is invoked. Since we invoke from the project root, `"tb/golden_primes.mem"` is correct. iVerilog silently zeroes memory on a missing file with NO error message, so this guard is essential -- without it, all 10006 candidates would appear non-prime and the testbench would produce wrong pass/fail results.

**Test tracking registers:**
```
integer errors;
integer test_count;
integer timeout_counter;
integer i;  // loop variable (allowed in testbench per INFRA-05 -- for loops OK in TB)
```

**DUT instantiation (must match exact port names from rtl/prime_engine.v):**
```
prime_engine #(.WIDTH(WIDTH)) dut (
    .clk        (clk),
    .rst        (rst),
    .start      (start),
    .candidate  (candidate),
    .done_ff    (done_ff),
    .is_prime_ff(is_prime_ff),
    .busy_ff    (busy_ff)
);
```

**Clock generation:**
```
initial clk = 0;
always #(CLK_PERIOD/2) clk = ~clk;
```

**Test task -- apply_and_check:**
```
task apply_and_check;
    input [WIDTH-1:0] cand;
    input             expected_prime;
    begin
        @(posedge clk);
        candidate = cand;
        start = 1'b1;
        @(posedge clk);
        start = 1'b0;
        // Wait for done_ff with timeout
        timeout_counter = 0;
        while (done_ff !== 1'b1 && timeout_counter < TIMEOUT_CYCLES) begin
            @(posedge clk);
            timeout_counter = timeout_counter + 1;
        end
        if (timeout_counter >= TIMEOUT_CYCLES) begin
            $display("FAIL: TIMEOUT candidate=%0d after %0d cycles", cand, TIMEOUT_CYCLES);
            errors = errors + 1;
        end else if (is_prime_ff !== expected_prime) begin
            $display("FAIL: candidate=%0d expected_prime=%0b got=%0b", cand, expected_prime, is_prime_ff);
            errors = errors + 1;
        end
        // Wait for done_ff to deassert before next test
        @(posedge clk);
    end
endtask
```

**Main test sequence:**
```
initial begin
    // Optional VCD dump for debugging
    $dumpfile("sim/prime_engine_tb.vcd");
    $dumpvars(0, prime_engine_tb);

    errors = 0;
    test_count = 0;
    rst = 1'b1;
    start = 1'b0;
    candidate = {WIDTH{1'b0}};

    // Hold reset for 4 cycles
    repeat(4) @(posedge clk);
    rst = 1'b0;
    @(posedge clk);

    // Sweep 2 through MAX_CANDIDATE
    for (i = 2; i <= MAX_CANDIDATE; i = i + 1) begin
        apply_and_check(i, golden[i]);
        test_count = test_count + 1;
        // Progress report every 1000 candidates
        if (i % 1000 == 0)
            $display("INFO: tested up to candidate %0d, errors so far: %0d", i, errors);
    end

    // Final report
    $display("---");
    $display("Tested %0d candidates (2..%0d)", test_count, MAX_CANDIDATE);
    if (errors == 0)
        $display("PASS: all %0d tests passed", test_count);
    else
        $display("FAIL: %0d errors out of %0d tests", errors, test_count);
    $display("---");
    $finish;
end
```

**CRITICAL details:**
- Use `$readmemb` (binary format, one bit per line), NOT `$readmemh`.
- The golden memory path in `$readmemb` should be `"tb/golden_primes.mem"` -- this is relative to the directory where `vvp` is invoked. Since we invoke from the project root, this is correct.
- The golden list load guard (`if (golden[2] !== 1'b1)`) MUST be present immediately after `$readmemb`. Without it, a missing .mem file silently produces all-zero memory, causing every candidate to appear non-prime with no error message.
- The `for` loop is in the testbench (tb/ file), NOT in a synthesis file. This is explicitly allowed by INFRA-05 ("permitted in testbenches only").
- Use `$display` for output, NOT `$error` (iVerilog compatibility per research pitfall 6).
- The `while` loop with timeout_counter avoids fork/join which can be problematic in some iVerilog versions.
- VCD dump is optional but useful for debugging; write to `sim/prime_engine_tb.vcd`.
- The testbench tests 10006 candidates (2 through 10007 inclusive). Expected: 1229 primes, 8777 composites.
