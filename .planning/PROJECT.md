# PrimeFPGA

## What This Is

A Nexys A7 FPGA-based prime number system that finds, validates, and tests prime numbers across four modes: find all primes up to N, find all primes within a time limit, check if a number is prime, and a hardware test mode against SD card reference data. Results are displayed on VGA (640x480, double-buffered in DDR2), with joystick + 7-segment entry for user input.

## Core Value

Correct, fast prime computation with a smooth VGA display — the 6k±1 algorithm must produce verified results with no screen tearing.

### Out of Scope

- Keypad input — joystick + 7SD chosen for number entry
- Non-SPI SD card modes (SDIO) — SPI is sufficient and simpler to implement
- Sieve of Eratosthenes — 6k±1 chosen for execution-time efficiency; large sieve would need prohibitive DDR2 bandwidth
- PMOD types other than joystick — spec requires at least one PMOD; joystick chosen

## Context

**Hardware platform:** Digilent Nexys A7-100T
- 100 MHz onboard oscillator
- 128 MB DDR2 (accessed via Xilinx MIG 7-series IP, AXI4 interface)
- VGA port: 4 bits per channel (R/G/B), 12-bit total color depth
- SD card slot (SPI-capable)
- 8x 7-segment displays (for number entry/debug)
- PMOD headers (joystick on PMOD JA or JB)

**Clock domains:**
- System/logic clock: 100 MHz (or MIG-derived)
- Pixel clock: ~25.175 MHz (for 640x480 @ 60Hz VGA)
- MIG UI clock: generated by MIG, typically 200 MHz reference + UI clock output
- SPI clock: ~400 kHz (SD card init) → up to 25 MHz (data transfer)

**Prime algorithm:** 6k±1 — check divisibility by 2 and 3, then test divisors of form 6k±1 up to √N. No for loops in synthesis; implemented as a state machine that iterates k.

**VGA double buffering:** Two full 640×480×16-bit framebuffers in DDR2 (~600 KB each). Write side updates the back buffer; display controller reads from the front buffer. Buffer swap is synchronized to VBlank.

**Build-small-first strategy:** Prime engine verified in simulation first, then integrated with DDR2, then VGA, then SD card, then full integration. Each module has its own testbench.

**Class design rules (CSEE 4280):**
- ANSI module port declarations
- No combinational logic at the top level
- Active-low signals: `_n` suffix; flip-flops: `_ff` suffix
- Blocking (`=`) and non-blocking (`<=`) assignments in separate `always` blocks
- No `for` loops in synthesis code
- Flip-flops: `always @(posedge clk)` with synchronous reset via `always @(*)`
- Reset handled in combinational logic, not as posedge sensitivity
- `default:` in all `case` statements; final `else` in all `if-else` chains
- All designs must have self-checking testbenches

## Constraints

- **Tech Stack**: Verilog only (no VHDL, no SystemVerilog for synthesis) — class requirement
- **IP**: Must use Xilinx MIG 7-series for DDR2 — required by spec
- **Toolchain**: Vivado for synthesis/implementation; iVerilog permitted for behavioral sim
- **Performance**: Execution time is the chosen optimization target — algorithm and architecture should maximize prime-finding throughput
- **Memory**: DDR2 latency (~50-100 ns) means prime storage writes are pipelined; VGA reads need predictable bandwidth
- **Grading**: Design not following class Verilog rules = 0.0 multiplier on functional score — rules are non-negotiable

## Key Decisions

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| 6k±1 trial division | Optimized for execution time; reduces trial divisors by ~67% vs naive; feasible as FSM without memory lookup tables | — Pending |
| SPI for SD card | Simpler than SDIO; sufficient read bandwidth for test file; well-documented for FPGA | — Pending |
| 16-bit pixel width (12-bit color + 4 pad) | Aligns DDR2 bursts to power-of-2 widths; matches Nexys A7 VGA color depth | — Pending |
| Double buffer in DDR2 (not BRAM) | BRAM insufficient for two 640×480 buffers (~1.2 MB total); DDR2 has capacity | — Pending |
| Build prime engine first | Verify correctness of core algorithm before integrating hardware peripherals | — Pending |
| Joystick + 7SD for input | Joystick satisfies PMOD requirement; 7SD gives clear numerical feedback during entry | — Pending |
